## Key findings

1. **All 165 `v1-*` functions map 1:1 to `functions/sdk-v1`** (NOT
   `packages/sdk-v1` — no such path; the package is `@levelup/functions-sdk-v1`
   at `functions/sdk-v1`). Deployed id `v1-<module>-<op>` derives mechanically
   from the nested export tree `v1.{identity,levelup,autograde,analytics}` in
   `functions/sdk-v1/src/index.ts`; the four module barrels are
   `src/{identity,levelup,autograde,analytics}.ts`. Zero drift in this
   direction.
2. **All but 3 of the 79 unprefixed functions map 1:1 to the 4 legacy
   codebases**, which are still active source under `functions/` (NOT moved to
   `old-deprecated/` — `old-deprecated/` contains app/product code, no
   deployable functions codebase; `firebase.json` still registers all 4 legacy
   codebases + sdk-v1).
3. **2 orphans:** `onClassDeleted` and `onStudentDeleted` — both `https`
   triggers. No export named either exists anywhere in the repo (functions/,
   packages/, old-deprecated/). These are leftovers from an older deploy
   generation; the current identity codebase replaced them with Firestore-update
   triggers `onClassArchived`/`onStudentArchived` (files
   `functions/identity/src/triggers/on-class-deleted.ts` /
   `on-student-deleted.ts` — the FILENAMES survive but the exports were
   renamed). Known DEP-1 gotcha: these https stubs needed delete-first handling
   during deploys. Safe-delete candidates.
4. **1 duplicate-export conflict:** `manageNotifications` is exported by BOTH
   `functions/identity` and `functions/levelup`. Only one function with that
   name can exist in the project; whichever codebase deployed last owns it, and
   a full deploy of the other codebase will fight over it (codebase-ownership
   error or steal). Must be resolved during cleanup.
5. **Reverse drift (defined in source, NOT deployed) — 3 functions:**
   - `v1-levelup-duplicateSpace` (functions/sdk-v1/src/levelup.ts) — wired
     post-deploy (web-SDK-migration parity gap)
   - `v1-autograde-saveExamQuestion` (functions/sdk-v1/src/autograde.ts) — same
   - `getItemForEdit` (functions/levelup/src/callable/get-item-for-edit.ts) —
     exported by legacy levelup index but absent from the deployed list (only
     `v1-levelup-getItemForEdit` is deployed)
6. **Auth v1 triggers** `onUserCreated`/`onUserDeleted` (gen-1
   `providers/firebase.auth` events) ARE actively sourced in
   `functions/identity` — legacy-active, not orphans. sdk-v1 has no auth-trigger
   equivalent, so they are load-bearing until claims bootstrap moves.
