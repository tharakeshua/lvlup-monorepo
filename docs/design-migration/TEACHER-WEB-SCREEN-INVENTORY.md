# teacher-web — Lyceum Screen Inventory (UI-1 Phase A)

Maps every current `apps/teacher-web` route/surface to its Lyceum prototype node
and per-screen spec. Prototype =
`old-deprecated/docs/rebuild-spec/design/build/app/web-teacher/App-Teacher.card.html`
(+ `ROUTE-TREE.md`, 57 nodes). Specs =
`old-deprecated/docs/rebuild-spec/design/{teacher,exams,spaces}/*.md`.

**Legend** — ✅ direct design coverage · 🟡 partial (adapt nearest spec) · ⚠️
GAP (no design → compose from system idioms per 00-FOUNDATION §5; do NOT invent
new tokens/variants).

Lanes refer to the Phase B migration order: **L1** exam lifecycle · **L2**
grading/submissions · **L3** people/classes · **L4** dashboard/analytics ·
**L5** shell/nav/auth.

## Gap policy (binding for all lanes)

A ⚠️/🟡 screen gets **system idioms only**: compose from 00-FOUNDATION §5
components with tokens from `@levelup/lyceum-preset`. No new colors, type sizes,
radii, shadows, or component variants. If a screen genuinely needs a new
primitive, escalate to the coordinator (foundation change), don't invent
locally.
