## Ordered Task List

### T1: Shared LazyImage Component [Feature] — S

Create a reusable image component with native lazy loading, error fallback, and
optional blur-up placeholder.

- **Files**: `packages/shared-ui/src/components/ui/LazyImage.tsx`,
  `packages/shared-ui/src/components/ui/index.ts`,
  `packages/shared-ui/src/index.ts`
- **Accept**: Component renders `<img loading="lazy" decoding="async">`, accepts
  `src`, `alt`, `width`, `height`, `className`, `fallback` props. Shows
  placeholder on error. Exported from shared-ui.

### T2: Add loading="lazy" to Existing Images [Feature] — M

Audit all `<img>` tags across all 5 apps and add `loading="lazy"` and
`decoding="async"` where images are below the fold. Keep hero/above-fold images
eager.

- **Files**: All components with `<img>` tags across 5 apps (LogoUploader,
  MaterialViewer, StoreDetailPage, SpacesListPage, StoreListPage, OrgSidebar,
  EditOrgGroupDialog, etc.)
- **Accept**: All below-fold images have `loading="lazy" decoding="async"`.
  Above-fold images explicitly set `loading="eager"`. No visual regressions.

### T3: Font Optimization [Feature] — S

Add `font-display: swap` to all font declarations and preload critical fonts in
index.html files.

- **Files**: All 5 `apps/*/index.html`, any CSS files with `@font-face`
  declarations, `packages/tailwind-config/index.js`
- **Accept**: No FOIT (flash of invisible text). Fonts render with system
  fallback then swap. Critical font files preloaded with
  `<link rel="preload" as="font" type="font/woff2" crossorigin>`.

### T4: Route Prefetch on Hover [Feature] — M

Implement route-level prefetching that triggers lazy import() on link
hover/focus, so the chunk is ready when navigation occurs.

- **Files**: `packages/shared-ui/src/components/ui/PrefetchLink.tsx` or
  `packages/shared-hooks/src/ui/usePrefetch.ts`, all 5 `apps/*/src/App.tsx` (add
  prefetch map)
- **Accept**: Hovering a sidebar/nav link for 100ms+ triggers the route's lazy
  import. No duplicate loads. Works with React.lazy(). Prefetch map per app
  defines route→import mappings.

### T5: PWA Install Banner Expansion [Integration] — S

Wire PWAInstallBanner into parent-web and teacher-web (consumer-facing apps).
Super-admin and admin-web are internal tools — skip.

- **Files**: `apps/parent-web/src/layouts/AppLayout.tsx`,

- **Accept**: PWAInstallBanner renders in parent-web and teacher-web layouts.
  Dismissible. Shows after 3+ page views.

## Risk Assessment

| Risk                                    | Mitigation                                                             |
| --------------------------------------- | ---------------------------------------------------------------------- |
| Route prefetch adds memory pressure     | Prefetch only on hover (not eagerly), limit concurrent prefetches to 3 |
| Image lazy loading causes layout shift  | Require width/height props on LazyImage, use aspect-ratio CSS          |
| Font preload increases initial download | Only preload 1-2 critical font weights, not all variants               |
| Bundle size limits too strict for CI    | Set initial thresholds at 120% of current baseline, tighten over time  |
