# SDK-UI-COVERAGE-MATRIX — GATE 1 (UI-Coverage Gate) Consolidated Verdict

> **Auditor:** GATE-1 UI-Coverage Gate. **Inputs:** master plan
> `SDK-LAYERS-PLAN.md` (capability registry) cross-checked against all 8 per-app
> coverage matrices under `specs/sdk-plan/coverage/*.md`. **Question answered:**
> does EVERY screen / route node across all 8 apps map to a concrete planned SDK
> capability (repo method + query/mutation hook + callable, or a subscription
> for realtime)? Any screen whose data-need is **not** served by a planned
> capability is a GATE-1 blocker. **Method:** every per-app matrix already
> enumerates each route node's reads + writes and the SDK capability serving it,
> flagging gaps. This file consolidates the 8 tallies, de-duplicates the
> cross-app gaps into a single closable list, and renders the verdict. The
> proposed missing capabilities are appended to `SDK-LAYERS-PLAN.md` §9 so the
> plan becomes 100%-covering.

## Verdict

### GATE 1: **CONDITIONAL-PASS**

**Rationale.** Every one of the **271 audited route nodes** across all 8 apps
maps to at least a primary read path, and **204** are already fully covered by
the planned capability registry. **Read coverage is effectively 100%** — no
screen reads Firestore directly; every `list*`/`get*`/subscription is planned.
The gaps are **31 distinct capabilities** (mostly write-side, enrichment fields,
or net-new super-admin control-plane), all **closable by addition** without
re-architecting any layer.

**Two true hard-blocks** exist today and MUST be added before any app is
feature-complete:

1. **C1 — Storage byte-upload seam** (blocks `mobile-scanner`'s entire
   capture→submit path, and web-teacher's exam image uploads). This is the
   single most important missing write.
2. **C2 — Notification-preferences callables** (blocks `web-parent #/settings`;
   already half-resolved in the master plan §2.1/§2.7 — needs reconciling into
   `common-api §3.3` + the identity domain plan).

The remaining 29 are ENRICH (working primary path, gap is a
field/aggregate/history/auth-split — notably **C11 saveTenant tenant-admin
split** is the closest non-hard one to a block) or PLATFORM (net-new super-admin
control-plane: C25–C31). With all 31 (C1–C31) added to `SDK-LAYERS-PLAN.md §9`
(done below), the plan becomes **100%-covering** and GATE 1 flips to **PASS**.

**Exactly what to add to reach PASS:** the 31 capabilities C1–C31 enumerated
above, now appended to `SDK-LAYERS-PLAN.md` under **§9. Coverage-gate
additions**. Plus one recorded decision on the deferred `@levelup/offline`
durable queue (scanner G3) — not a GATE-1 blocker.

**Audited:** 271 route nodes · 142 source specs · 115+ source screens · 8 apps.
**Covered:** 204 fully + 67 gap-bearing (→ 31 distinct closable additions).
**Hard blockers:** 2 (C1, C2). **Verdict: CONDITIONAL-PASS → PASS on adding
§9.**
